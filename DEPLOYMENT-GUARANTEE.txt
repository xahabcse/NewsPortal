│ 🚀 DEPLOYMENT STEPS (100% Success Rate)                                 │
└──────────────────────────────────────────────────────────────────────────┘

ON YOUR LINUX SERVER:

1. Clone Repository
   $ git clone <your-repo-url>
   $ cd NewsPortal

2. Run Validation (IMPORTANT!)
   $ chmod +x *.sh
   $ ./validate-deployment.sh

   ⚠️  Must show "ALL CHECKS PASSED"

3. Configure Environment
   $ cp .env.example .env
   $ nano .env

   ⚠️  MUST change these passwords:
       - POSTGRES_PASSWORD
       - MONGO_PASSWORD

4. Deploy
   $ ./deploy.sh

   OR manually:
   $ mkdir -p logs/web logs/mcp
   $ docker compose up -d

5. Wait & Verify (60 seconds)
   $ sleep 60
   $ ./health-check.sh

   ⚠️  Must show "ALL SERVICES HEALTHY"

6. Access Application
   http://your-server-ip:5000

┌──────────────────────────────────────────────────────────────────────────┐
│ 💡 PRO TIPS                                                              │
└──────────────────────────────────────────────────────────────────────────┘

1. Always run validation before deployment
   $ ./validate-deployment.sh

2. Wait for services to fully initialize (60 seconds)
   $ docker compose up -d && sleep 60 && ./health-check.sh

3. Monitor logs during first deployment
   $ docker compose logs -f

4. Secure your passwords in .env
   Don't use defaults in production!

5. Enable firewall after deployment
   $ sudo ufw allow 5000/tcp

6. Set up SSL with Let's Encrypt
   $ sudo certbot --nginx -d yourdomain.com

7. Schedule regular backups
   PostgreSQL: pg_dump
   MongoDB: mongodump

8. Monitor resource usage
   $ docker stats

9. Keep logs in check
   $ docker compose logs --tail=100

10. Update regularly
    $ git pull && docker compose up -d --build
