# Project Rules for Claude Code

## Communication
- Always respond in **Bangla**. Use English only for technical keywords, code references, file names, and programming terms.

## Workflow
- For every task, **first create a plan** and present it to the user for approval before starting implementation.
- Never start coding without an approved plan.

## Verification
- After implementing any task, **build and run** the project to verify all changes are applied correctly.
- Ensure the changes do not break any other part of the application.
- Test affected areas before marking a task as complete.

## Git Commits
- Do **NOT** add any co-author line (e.g., `Co-Authored-By`) in commit messages.
- Do **NOT** add any "supported by Claude" or similar attribution in commits.
- Keep commit messages clean and concise.
