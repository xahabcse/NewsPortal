================================================================================
                    ১. MCP Server কী এবং এটি কীভাবে কাজ করে
================================================================================

MCP এর পূর্ণরূপ হলো Model Context Protocol। এটি Anthropic দ্বারা তৈরি একটি 
ওপেন প্রোটোকল যা AI মডেল (যেমন Claude) কে বাহ্যিক সার্ভিস, টুলস এবং ডেটা 
সোর্সের সাথে সংযুক্ত করতে সাহায্য করে।

সহজ ভাষায়:
-----------
MCP কে আপনি একটি "সেতু" বা "সংযোগকারী" হিসেবে ভাবতে পারেন। ধরুন, Claude 
একটি ঘরে বসে আছে, কিন্তু বাইরের বিভিন্ন সেবা যেমন Canva, Gmail, Asana, 
Salesforce ইত্যাদির সাথে কথা বলতে চায়। MCP সেই দরজা বা রাস্তা তৈরি করে 
যার মাধ্যমে Claude এই বাহ্যিক সেবাগুলোর সাথে যোগাযোগ করতে পারে।

MCP কীভাবে কাজ করে:
-------------------
প্রথম ধাপ: ব্যবহারকারী তাদের পছন্দের সার্ভিস এর সাথে MCP সংযোগ স্থাপন করেন।
দ্বিতীয় ধাপ: যখন ব্যবহারকারী Claude কে কোনো কাজ করতে বলেন, Claude সেই 
             MCP সার্ভারের মাধ্যমে সংশ্লিষ্ট সার্ভিসে অনুরোধ পাঠায়।
তৃতীয় ধাপ: বাহ্যিক সার্ভিস কাজটি সম্পন্ন করে এবং ফলাফল ফেরত পাঠায়।
চতুর্থ ধাপ: Claude সেই ফলাফল ব্যবহারকারীকে দেখায়।

MCP এর সুবিধা:
- একীভূত অভিজ্ঞতা: একই জায়গা থেকে বিভিন্ন অ্যাপ ও সার্ভিস ব্যবহার করা যায়
- স্বয়ংক্রিয়তা: Claude আপনার হয়ে বিভিন্ন প্ল্যাটফর্মে কাজ করতে পারে
- নিরাপত্তা: প্রতিটি সংযোগ ব্যবহারকারীর অনুমতি সাপেক্ষে কাজ করে

================================================================================
            ২. .NET ডেভেলপারদের জন্য MCP Server শেখার প্রজেক্ট
================================================================================

বিগিনার লেভেল প্রজেক্ট:
-----------------------

১. Simple Calculator MCP Server
   একটি বেসিক ক্যালকুলেটর সার্ভার তৈরি করুন যেখানে যোগ, বিয়োগ, গুণ, ভাগ 
   এর টুলস থাকবে।

২. Weather Information Server
   একটি আবহাওয়া তথ্য সার্ভার তৈরি করুন যা কোনো পাবলিক API থেকে ডেটা 
   এনে দেখাবে।

৩. To-Do List Manager
   একটি সাধারণ টাস্ক ম্যানেজার যেখানে টাস্ক তৈরি, পড়া, আপডেট এবং ডিলিট 
   করার সুবিধা থাকবে।

ইন্টারমিডিয়েট লেভেল প্রজেক্ট:
-----------------------------

৪. File System Explorer
   একটি ফাইল সিস্টেম এক্সপ্লোরার যেখানে ফাইল পড়া, লেখা, ফোল্ডার তৈরি 
   এবং সার্চ করার সুবিধা থাকবে।

৫. Database Query Assistant
   SQL Server বা SQLite ডেটাবেসের সাথে সংযুক্ত একটি সার্ভার তৈরি করুন 
   যা ডেটা কুয়েরি করতে পারবে।

৬. Email Reader/Sender
   SMTP এবং IMAP/POP3 ব্যবহার করে ইমেইল পড়া ও পাঠানোর সার্ভার।

৭. GitHub Repository Manager
   GitHub API ব্যবহার করে রিপোজিটরি তালিকা দেখা, ইস্যু তৈরি করা, পুল 
   রিকোয়েস্ট দেখার সুবিধা সহ সার্ভার।

অ্যাডভান্সড লেভেল প্রজেক্ট:
---------------------------

৮. Document Management System
   PDF, Word, Excel ফাইল পড়া, তৈরি এবং এডিট করার সার্ভার।

৯. Azure Services Integration Server
   Azure Blob Storage, Azure SQL, Azure Functions ইত্যাদির সাথে 
   সংযুক্ত একটি সার্ভার।

১০. Enterprise CRM Connector
    Dynamics 365 বা কাস্টম CRM এর সাথে সংযুক্ত সার্ভার।

১১. Multi-Service Orchestrator
    একাধিক সার্ভিস একসাথে ম্যানেজ করার সার্ভার।

                    ৪. ডেটাবেস ও ক্যাশ নির্বাচন

ডেটাবেস অপশন:
--------------

১. SQL Server Express (ফ্রি) - প্রথম পছন্দ
   - .NET এর সাথে সবচেয়ে ভালো কাজ করে
   - Visual Studio তে বিল্ট-ইন সাপোর্ট
   - ১০ GB পর্যন্ত ফ্রি

২. PostgreSQL (ফ্রি) - দ্বিতীয় পছন্দ
   - সম্পূর্ণ ফ্রি, কোনো লিমিট নেই
   - JSON ডেটা টাইপ সাপোর্ট করে
   - Full-Text Search বিল্ট-ইন

৩. SQLite - ছোট প্রজেক্টের জন্য
   - কোনো ইনস্টলেশন লাগে না
   - ডেভেলপমেন্ট ও টেস্টিং এর জন্য দারুণ

ক্যাশিং অপশন:
--------------

১. In-Memory Cache (IMemoryCache) - প্রথম পছন্দ
   - .NET এ বিল্ট-ইন
   - সেটআপ অত্যন্ত সহজ
   - ছোট থেকে মাঝারি প্রজেক্টের জন্য পারফেক্ট

২. Redis - দ্বিতীয় পছন্দ
   - ডিস্ট্রিবিউটেড ক্যাশিং
   - অ্যাপ রিস্টার্ট হলেও ক্যাশ থাকে
   - খুব দ্রুত

ধাপ ১: সার্ভার ল্যাপটপ প্রস্তুত করুন
-------------------------------------
# লিনাক্স আপডেট করুন
sudo apt update && sudo apt upgrade -y

sudo systemctl enable ssh
sudo systemctl start ssh

# সার্ভারের IP অ্যাড্রেস বের করুন
ip addr show

ধাপ ২: স্ট্যাটিক IP সেট করুন
-----------------------------

network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses:
        - 192.168.1.20/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

sudo netplan apply

ধাপ ৪: Docker Compose ফাইল তৈরি করুন
--------------------------------------
mkdir -p ~/newsportal
cd ~/newsportal
nano docker-compose.yml

version: '3.8'

services:
  # PostgreSQL - মেইন ডেটাবেস
  postgres:
    image: postgres:15-alpine
    container_name: newsportal-db
    restart: always
    environment:
      POSTGRES_USER: newsadmin
      POSTGRES_PASSWORD: YourSecurePassword123
      POSTGRES_DB: newsportal
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    deploy:
      resources:
        limits:
          memory: 512M

  # MongoDB - ইমেজ স্টোরেজ
  mongodb:
    image: mongo:4.4
    container_name: newsportal-mongodb
    restart: always
    environment:
      MONGO_INITDB_ROOT_USERNAME: mongouser
      MONGO_INITDB_ROOT_PASSWORD: MongoPassword123
    volumes:
      - mongodb_data:/data/db
    ports:
      - "27017:27017"
    deploy:
      resources:
        limits:
          memory: 512M

  # Redis - ক্যাশিং
  redis:
    image: redis:alpine
    container_name: newsportal-cache
    restart: always
    ports:
      - "6379:6379"
    deploy:
      resources:
        limits:
          memory: 128M

volumes:

news_articles:
- id (PK)

- source_id (FK), category_id (FK)
- source_url, original_image_url
- mongo_image_id, mongo_thumb_id
- author, published_at, fetched_at
- view_count, is_featured, is_active

news_sources:
- id (PK)
- name, slug, base_url, logo_url

- rss_feed_url, api_endpoint, api_key

scraping_configs:
- id (PK), source_id (FK)
- list_page_url
- article_selector, title_selector, content_selector
- image_selector, date_selector, author_selector

categories:
- id (PK)
- name, name_bn, slug, description
- icon, color, sort_order, is_active
